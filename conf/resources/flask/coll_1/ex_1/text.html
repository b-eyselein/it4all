In dieser Aufgabe soll die Loginfunktionalität für eine Webseite implementiert werden. Dazu soll den Nutzern außerdem die Möglichkeit der Registrierung geboten werden. Um den Login über verschiedene Requests hinweg verfügbar zu machen, soll der Nutzername in der Session gespeichert werden.

Es sollen drei Templates erstellt werden, die alle drei das gegebene Basis-Template base.html erweitern und den Inhalt im Block content rendern::
  • index.html: Dieses Template soll nur eine h1-Überschrift mit dem Textinhalt Startseite anzeigen.
  • login.html: Dieses Template rendert ein typisches Login-Formular. Dieses soll als Daten einen Benutzernamen (name := 'name') und ein Passwort (name := 'password') erfassen und diese Daten per POST-Request an die URL /login senden.
  • register.html: Dieses Template rendert ein typisches Registrierungs-Formular. Dieses soll als Daten einen Benutzernamen (name := 'name') und zwei mal dasselbe Passwort (name := 'password' und name := 'passwordRepeat') erfassen und diese Daten per POST-Request an die URL /register senden.
Es sollen insgesamt Routes für 3 URLs implementiert werden:
  • /: Wenn diese URL aufgerufen wird, soll überprüft werden, ob ein Nutzer eingeloggt ist. Falls nicht, soll eine Weiterleitung zur URL /login stattfinden. Ansonsten soll das Template index.html angezeigt werden.
  • /login: Wenn diese URL mit der Methode GET aufgerufen wird, soll das Template login.html gerendert werden. Dieses sendet die Formulardaten per POST-Request an die selbe URL. Daher sollen in diesem Fall die Formulardaten ausgelesen und der Nutzer validiert werden. Bei einer erfolgreichen Validierung wird der Nutzername in der Session gespeichert und es soll eine Weiterleitung auf die URL / stattfinden. Ansonsten soll wiederum das Template login.html angezeigt werden.
  • /register: Wenn diese URL mit der Methode GET aufgerufen wird, soll das Template register.html angezeigt werden. Dieses Formular senden die Daten per POST-Request an dieselbe URL. In diesem Fall sollen die Formulardaten ausgelesen und validiert werden. Bei erfolgreicher Validierung soll der Nutzer registriert werden, ansonsten soll wieder das Template register.html angezeigt werden.
